أهلاً بك يا صديقي المبدع في محطة ذكية جداً من رحلة احتراف الويب. هل سئمت يوماً من البحث في آلاف الأسطر من الكود لتغيير درجة لون معينة؟ أو هل شعرت بالملل من تكرار نفس قيم المسافات والخطوط في كل "Selector" تكتبه؟ إذا كانت إجابتك نعم، فأنت في المكان الصحيح.

اليوم سنتحدث عن **كل ما تحتاج لمعرفته حول متغيرات CSS**، أو ما يعرف تقنياً بـ **CSS Custom Properties**. هذه الميزة ليست مجرد "رفاهية"، بل هي الأداة التي ستحول كودك من "فوضى مكررة" إلى "نظام ذكي ومنظم". في هذا الدليل، سنتعلم **كيف يتم تعريف المتغيرات في لغة CSS؟** وسنكشف أسرار **استخدام متغيرات CSS كالمحترفين**. استعد، لأننا سنجعل كودك أكثر أناقة وسهولة في التعديل مما تخيلت يوماً!

---

## الجزء الأول: تمهيد عن ثورة المتغيرات في CSS

قبل ظهور المتغيرات، كان المطورون يلجؤون للغات مساعدة مثل Sass أو Less ليتمكنوا من استخدام المتغيرات. لكن الآن، أصبحت هذه الميزة متوفرة "محلياً" داخل CSS.

المتغير ببساطة هو "وعاء" تعطي له اسماً (مثل `--main-color`) وتخزن فيه قيمة (مثل `#3498db`). بدلاً من كتابة الكود اللوني في كل مكان، تنادي فقط على اسم الوعاء.

هذا الدليل مصمم ليكون مرجعك لفهم:

* **CSS Variables** وكيفية عملها في المتصفحات الحديثة.
* الطريقة الصحيحة للتعريف باستخدام نطاق `:root`.
* كيفية بناء "ثيمات" (Themes) متغيرة مثل الوضع الليلي (Dark Mode) بضغطة زر.
* استراتيجيات تنظيم الأكواد للمشاريع الكبيرة.

بحلول نهاية هذا المقال، ستتوقف عن كتابة القيم الثابتة، وستبدأ في بناء "أنظمة تصميم" (Design Systems) مرنة وقابلة للتطوير.

---

## الجزء الثاني: محتويات دليل المتغيرات الشامل

لقد رتبنا لك المحتوى ليكون رحلة تعليمية منطقية:

1. **أساسيات التعريف:** كيف يتم تعريف المتغيرات في لغة CSS؟
2. **الاستدعاء الذكي:** استخدام وظيفة `var()` والقيم الاحتياطية.
3. **نطاق العمل (Scope):** الفرق بين المتغير العام (Global) والمحلي (Local).
4. **المزايا والفوائد:** لماذا يجب أن تستخدم المتغيرات الآن؟
5. **تطبيقات احترافية:** بناء نظام ألوان ووضع ليلي متكامل.

> **ملاحظة:** المتغيرات في CSS "ديناميكية"، وهذا يعني أنه إذا تغيرت قيمة المتغير عبر JavaScript، سيتحدث التصميم فوراً في المتصفح دون إعادة تحميل الصفحة.

---

## الجزء الثالث: كيف يتم تعريف المتغيرات في لغة CSS؟

لتعريف متغير، نستخدم بادئة مكونة من "شرطتين" (`--`) متبوعة باسم من اختيارك.

### 1. التعريف في الجذر (:root)

لجعل المتغير متاحاً في كل أرجاء الموقع، نقوم بتعريفه داخل Selector يسمى `:root` (وهو يمثل عنصر `<html>`).

```css
:root {
  --primary-color: #3498db;
  --secondary-color: #2ecc71;
  --main-font: 'Cairo', sans-serif;
  --main-padding: 20px;
}

```

### 2. طريقة الاستدعاء باستخدام `var()`

بمجرد تعريف المتغير، يمكنك استخدامه في أي مكان داخل الكود باستخدام الوظيفة `var()`.

```css
button {
  background-color: var(--primary-color);
  padding: var(--main-padding);
  font-family: var(--main-font);
}

```

## الجزء الرابع: استخدام متغيرات CSS كالمحترفين: دليل المبتدئين

المحترفون لا يضعون المتغيرات عشوائياً، بل يتبعون نظاماً يجعل الكود "نظيفاً" وقابلاً للقراءة.

### 3. ميزة القيمة الاحتياطية (Fallback Value)

ماذا لو فُقد المتغير أو لم يتم تعريفه؟ يمكنك وضع قيمة احتياطية داخل وظيفة `var()`.

```css
.card {
  color: var(--text-color, #333); /* إذا لم يجد المتغير، سيستخدم اللون الرمادي */
}

```

### 4. النطاق المحلي (Local Scope)

أحياناً تريد متغيراً يعمل فقط داخل جزء معين من الصفحة (مثل الـ Sidebar). يمكنك تعريفه داخل الـ Selector الخاص به.

```css
.sidebar {
  --sidebar-bg: #2c3e50;
  background-color: var(--sidebar-bg);
}
/* المتغير --sidebar-bg غير متاح خارج السايدبار */

```

## الجزء الخامس: لماذا نحب CSS Variables؟ (المزايا)

استخدام المتغيرات ليس مجرد موضة، بل هو ضرورة تقنية:

| الميزة | الفائدة العملية |
| --- | --- |
| **سهولة التعديل** | تغيير اللون في مكان واحد (root) يغيره في كامل الموقع فوراً. |
| **وضوح الكود** | `--brand-red` أوضح بكثير للقارئ من الكود `#ff4d4d`. |
| **التفاعلية** | يمكن تغيير قيم المتغيرات بسهولة باستخدام JavaScript لصنع تأثيرات حية. |
| **دعم الـ Responsive** | يمكنك تغيير قيمة المتغير داخل Media Query واحدة لتتغير أحجام كل العناصر. |

---

## الجزء السادس: تطبيقات عملية (نظام الثيمات)

أقوى مثال على قوة **CSS Variables** هو إنشاء "الوضع الليلي" (Dark Mode).

```css
:root {
  --bg-color: #ffffff;
  --text-color: #000000;
}

[data-theme='dark'] {
  --bg-color: #1a1a1a;
  --text-color: #f0f0f0;
}

body {
  background-color: var(--bg-color);
  color: var(--text-color);
  transition: all 0.3s ease; /* لإضافة نعومة عند تغيير الثيم */
}

```

---

## الجزء السابع: نصائح ذهبية لتنظيم المتغيرات

* **التسمية المنطقية:** لا تسمي المتغير `--blue` (لأنك قد تغيره لاحقاً ليكون أحمر)، بل سمه حسب وظيفته مثل `--main-brand-color`.
* **متغيرات للمسافات:** استخدم المتغيرات للـ `gap` والـ `padding` لضمان تناسق المسافات في كل الصفحة.
* **الدمج مع الحسابات:** يمكنك استخدام `calc()` مع المتغيرات:
`width: calc(100% - var(--sidebar-width));`

---

## الجزء الثامن: خلاصة دليل متغيرات CSS

لقد اكتشفت اليوم كيف أن **متغيرات CSS** هي المفتاح السحري لكتابة كود نظيف واحترافي.

1. **التعريف:** يتم في الـ `:root` ببادئة `--`.
2. **الاستخدام:** يتم عبر وظيفة `var()`.
3. **القوة:** تكمن في القدرة على تغيير شكل الموقع بالكامل من مكان واحد.

---

### الأسئلة الشائعة

> **س: هل تدعم المتصفحات القديمة مثل Internet Explorer المتغيرات؟**
> ج: للأسف لا، ولكن جميع المتصفحات الحديثة (Chrome, Firefox, Safari, Edge) تدعمها بنسبة 100%. للمتصفحات القديمة جداً، قد تحتاج لاستخدام PostCSS كحل بديل.

> **س: هل يمكنني تخزين صور في المتغيرات؟**
> ج: نعم! يمكنك تخزين الروابط `url()` داخل متغير واستدعائها في `background-image`.

**الخلاصة:**
